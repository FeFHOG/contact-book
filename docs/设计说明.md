# 通讯录管理系统设计说明

## 一、需求分析

系统需要管理手机通讯录和手机卡通讯录。手机联系人包含姓名和电话号码；手机卡联系人包含姓名、电话号码、籍贯和 QQ。系统支持添加、删除、修改、查询、浏览、复制和移动联系人，并通过文本文件保存数据。

## 二、类设计

`ContactCard` 表示基础联系人，包含姓名和电话号码。

`ContactPhone` 继承 `ContactCard`，在基础信息上增加籍贯和 QQ。

`ContactBookInterface` 是通讯录抽象接口，定义删除、查找、显示和获取数量等虚函数，用于体现多态。

`ContactTree` 是 AVL 平衡二叉搜索树模板，用电话号码作为关键字保存联系人，支持插入、删除、查找和中序遍历。

`ContactBookCard` 管理手机联系人，内部使用 `ContactTree<ContactCard>` 保存数据。

`ContactBookPhone` 管理手机卡联系人，内部使用 `ContactTree<ContactPhone>` 保存数据。

`User` 类拥有一个手机通讯录和一个手机卡通讯录，负责完成用户菜单中的各项操作。

`contact_validator` 模块负责校验电话号码格式。系统要求电话号码为 6 到 20 位数字，可以带一个开头的 `+` 号。

## 三、文件存储

手机联系人保存在 `data/phone.txt`。

手机卡联系人保存在 `data/card.txt`。

程序启动时从文件读取数据，程序退出时由通讯录对象的析构函数自动写回文件。

## 四、查找算法设计

通讯录内部使用 AVL 平衡二叉搜索树保存联系人。树中每个节点保存一个联系人对象，并以电话号码作为排序关键字。插入和删除节点后，系统通过左旋、右旋等操作维护树的平衡，使树高保持在较低水平。按电话号码查找、添加和删除联系人在平均和稳定情况下都可以达到 `O(log n)` 复杂度。

按姓名查询时使用 KMP 字符串匹配算法。用户输入姓名关键字后，系统可以匹配完整姓名，也可以匹配姓名中的一部分。例如输入“张”可以查询到“张三”。KMP 通过前缀表减少重复比较，时间复杂度为 `O(n + m)`，其中 `n` 是联系人姓名长度，`m` 是查询关键字长度。

KMP 算法被单独封装在 `search_algorithm.h` 和 `search_algorithm.cpp` 中，手机通讯录和手机卡通讯录都调用同一个 `kmpContains` 函数，避免重复代码，也使程序结构更清晰。

## 五、面向对象特性

继承：`ContactPhone : public ContactCard`。

多态：`ContactBookCard` 和 `ContactBookPhone` 继承 `ContactBookInterface`，重写虚函数。

运算符重载：重载 `operator>>` 和 `operator<<`，用于联系人对象的输入输出。

函数重载/同名函数：两个通讯录类都有 `addContact`、`deleteContact`、`display` 等同名成员函数。

输入校验：输入联系人电话号码时，系统会调用 `isValidPhoneNumber` 判断格式是否合法，不合法则提示用户重新输入。
